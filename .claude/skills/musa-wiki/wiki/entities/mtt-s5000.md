# MTT S5000

The MTT S5000 is Moore Threads' flagship data-center GPU, based on the **MP31** architecture (MUSA 3.x). It targets LLM training and inference workloads.

## Specs (Indicative)

| Spec | Value | Notes |
|------|-------|-------|
| Architecture | MP31 | MUSA 3.x |
| MPs (SMs) | ~128 | Query via `musaDevAttrMultiprocessorCount` |
| Warp size | 32 | Same as NVIDIA CUDA |
| Max threads/SM | 2048 | |
| FP32 peak | ~40 TFLOPS | |
| FP16 Tensor Core peak | ~160 TFLOPS | |
| BF16 Tensor Core peak | ~160 TFLOPS | |
| Memory | 64 GB GDDR6 (typical) | |
| Memory bandwidth | ~1.2 TB/s | |
| L2 cache | ~96 MB | |
| Shared mem/SM | 48-96 KB | Configurable |
| TDP | ~250 W | |

> **Caveat**: Specifications vary by board design and SKU. Always query via `musaGetDeviceProperties` on the actual hardware.

## Architecture Notes

- **Warp size = 32**: Same as NVIDIA CUDA. Code written for warp=32 works without modification.
- **Tensor Cores**: Available in FP16, BF16, TF32, INT8. See [[tensor-cores]].
- **Cluster support**: Yes — supports Thread Block Clusters for distributed shared memory. See [[cluster-memory]].
- **L2 persistence**: Supported. See [[l2-cache-management]].
- **Green Context**: Supported. See [[green-context]].

## Compile Target

```bash
mcc -arch=mp31 kernel.mu -lmusart -o kernel
```

Or generate a fatbin supporting multiple architectures:

```bash
mcc -arch=mp21 -arch=mp22 -arch=mp31 kernel.mu -o kernel.fatbin
```

## Compared to Other MUSA GPUs

| Spec | MTT S5000 (MP31) | MTT M1000 (MP22) | MTT S4000 (MP21) |
|------|-------------------|-------------------|-------------------|
| Warp size | 32 | 128 | 128 |
| Target | Data center | Workstation | Workstation |
| FP16 TC | ~160 TFLOPS | ~10 TFLOPS | ~10 TFLOPS |
| Memory BW | ~1.2 TB/s | ~256 GB/s | ~256 GB/s |

The warp size difference is the biggest porting concern when targeting multiple MUSA architectures. Code that uses `__shfl_*_sync` or assumes `warpSize == 32` may break on MP21/MP22.

## Comparison to NVIDIA Hardware

| Spec | MTT S5000 | NVIDIA H100 (SXM) |
|------|-----------|---------------------|
| FP16 TC | ~160 TFLOPS | ~989 TFLOPS (dense) |
| Memory BW | ~1.2 TB/s | ~3.35 TB/s |
| Warp size | 32 | 32 |

> Note: This is a high-level comparison. The MTT S5000 is positioned as a cost-effective alternative in the domestic Chinese market, not a direct H100 competitor on raw throughput.

## Discovery

```cpp
int dev; musaGetDevice(&dev);
musaDeviceProp prop; musaGetDeviceProperties(&prop, dev);
if (prop.major == 3 && prop.minor == 1) {
    // MP31 (MTT S5000)
    printf("Running on MTT S5000\n");
    printf("MPs: %d, warpSize: %d\n", prop.multiProcessorCount, prop.warpSize);
}
```

## Optimization Notes for S5000

- **Tensor Cores are highly effective**: target FP16/BF16 MMA for matrix workloads.
- **L2 is large**: consider L2 persistence for hot working sets up to ~80 MB.
- **128 MPs**: large grids (≥ 128 blocks) maximize occupancy.
- **GDDR6 bandwidth**: ~1.2 TB/s — coalescing and shared mem tiling are critical.
