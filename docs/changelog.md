## v3.1

<!-- 原位置：design-proposal.md §1.3 v3.1 重大变更说明 -->

相比 v3.0，v3.1 本次修订的核心变更（针对赛题加分项深度优化）：

| 变更 | 原因 | 影响 |
|------|------|------|
| **新增 Router 自学习引擎** | **v3.0 的"即时偏好引擎"只是偏好记忆注入，不是赛题要求的"动态优化分发策略"** | **真正实现路由置信度评分 + 双层路由 + 用户修正反馈 + 路由策略动态调整** |
| **新增路由置信度评分** | **支撑双层路由和自学习** | **路由模型启用 logprobs，输出 top-1 路由 + 置信度** |
| **新增双层路由机制** | **低置信度时主动询问用户，避免误路由** | **L1 自动路由 + L2 确认路由，建立用户信任** |
| **新增 Subagent 市场机制** | **v3.0 只是"合入 MTClaw 目录"，不是赛题要求的"市场"** | **CLI + registry 索引 + 安装/卸载/更新，含官方预置 + 社区贡献** |
| **即时偏好引擎降级为辅助** | **让位给 Router 自学习引擎作为核心** | **即时偏好仍负责内容生成偏好，路由策略优化由 Router 自学习引擎负责** |

## v3.0

<!-- 原位置：design-proposal.md §1.4 v3.0 重大变更说明 -->

相比 v2.0，v3.0 的核心变更：

| 变更 | 原因 | 影响 |
|------|------|------|
| Subagent 从 8 个缩减到 5 个，再调整为 5 个（砍 Bash 换日程与任务） | 砍掉 WebFetch/WebSearch/DataAnalysis（上游 LLM 兜底覆盖）；砍掉 Bash（赛题不面向开发者，MTClaw builtin 已覆盖文件操作）；新增日程与任务（赛题推荐方向，面向知识工作者） | 工具数从 24 降到 16，路由准确率提升 |
| 砍掉插件系统 | 比赛不需要，直接写死配置 | 省 1 周开发时间 |
| 砍掉知识图谱 | 小数据量上无实际价值 | 省 3-5 天 |
| 砍掉模型 fallback 链 | 比赛环境不会挂 | 省 2 天 |
| 砍掉上下文压缩器 | MTClaw 已有 fr_context_history | 省 2-3 天 |
| 砍掉多搜索后端 fallback | DuckDuckGo 够用 | 省 2 天 |
| 反思引擎改为即时偏好引擎 | 演示中能即时展示"进化"效果 | 演示可跑通 |
| 所有性能数字标注来源 | 诚实化，区分"实测"和"目标" | 应对评委追问 |
| 工具数 16 个（含 builtin 共 21） | LLM function calling 在 <15 工具时准确率最高 [推测]，16 个略超但接近，靠 5 个 Subagent 清晰划分保持路由精度 | 路由更准 |
| **加回轻量路由追踪面板** | **赛题加分项明确要求"可视化路由追踪"** | **单 HTML 文件 + 轮询 API，1 天开发量** |
| **商业价值扩写为 8 个子节** | **对齐赛题"生态和商业价值"6 个子维度** | **真实场景/产品化/商业模式/生态适配/可复用/示范带动** |
| **加分项对齐赛题原文** | **赛题明确列出 3 个加分项** | **Router 自学习 + 可视化路由追踪 + 开箱即用** |
| **评分矩阵对齐赛题维度** | **赛题原文评分维度** | **快/准/狠/生态商业价值/加分项** |
| **从"无损接入"改为"源码合入 MTClaw"** | **官方希望最终代码提交到 MTClaw 开源仓库** | **用 MTClaw 自带安装脚本一键安装，不独立维护安装体系** |
